# EuroPulse Week 5 — Build Plan

## Objective
Refactor the Streamlit dashboard into a professional four-tab layout, integrate
LLM-powered narrative generation for regimes and risk, and add a research-chat
interface backed by the ChromaDB news archive.

## Deliverables

### 1. `europulse/llm/synthesizer.py`
New module exposing three high-level LLM wrappers:
- `generate_regime_narrative(regime_df)` — queries the LLM for a plain-English
  interpretation of the latest composite macro regime.
- `generate_risk_summary(risk_df)` — queries the LLM for a portfolio risk
  summary given volatility, drawdown, Sharpe, and beta metrics.
- `chat_with_data(query, top_k)` — performs RAG retrieval over the news
  collection via `query_similar`, injects the top-k articles into the prompt,
  and returns a concise, evidence-based answer.

### 2. `europulse/ui/app.py`
Complete refactor from a seven-page sidebar-radio layout to a four-tab dashboard
with persistent sidebar controls:
- **Sidebar** — ticker multiselect and date-range picker that filter all tabs.
- **Tab 1: Macro & Regimes** — composite regime line chart, latest regime
  table, and AI narrative card generated by `generate_regime_narrative`.
- **Tab 2: Risk & Correlation** — risk metrics table (volatility, max
  drawdown, Sharpe, beta to SPY), correlation heatmap powered by
  `correlation_matrix`, and AI risk summary card generated by
  `generate_risk_summary`.
- **Tab 3: Forecast** — model selector (Exponential Smoothing vs Linear
  Regression), ticker selectbox, horizon slider, and historical + forecast
  chart with confidence bands.
- **Tab 4: Research Assistant** — chat interface using `st.chat_input` and
  `st.chat_message`, backed by `chat_with_data` and `st.session_state` for
  conversation history.

### 3. `tests/test_synthesizer.py`
Unit tests for all three synthesizer functions. LLM and RAG calls are mocked
with `unittest.mock.patch` so tests run offline and fast.

### 4. `tests/test_ui.py`
Import-safety test that sets `config.DB_PATH` to a non-existent file, imports
`europulse.ui.app`, and asserts the presence of the `load_data` function.

## Dependencies
- Existing DuckDB, ChromaDB, Plotly, and Streamlit infrastructure.
- `europulse.llm.client.query_llm` for LLM calls.
- `europulse.rag.embeddings.query_similar` for RAG retrieval.
- `europulse.analysis.regimes.detect_regimes` for regime detection.
- `europulse.analysis.risk.beta_to_benchmark`, `correlation_matrix`,
  `rolling_volatility`, `max_drawdown`, `sharpe_ratio` for risk analytics.
- `europulse.analysis.forecast.exp_smooth_forecast` and `linear_forecast`
  for the forecast tab.

## Verification Criteria
- All existing tests continue to pass.
- New synthesizer tests pass with mocked LLM/RAG.
- New UI import test passes.
- `streamlit run europulse/ui/app.py` launches without import errors.
